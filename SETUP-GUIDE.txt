  STEP 4 — Get YouTube OAuth Tokens (one-time, lasts forever)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  This system uses YouTube's
  official API with OAuth refresh tokens. These tokens last
  FOREVER — you do this once and never again.

  You need 3 things: Client ID, Client Secret, Refresh Token.

  ─── 4A. Create a Google Cloud Project ───

    a. Go to https://console.cloud.google.com/
    b. Click the project dropdown at the top → "New Project"
    c. Name it anything (e.g., "yt-shorts-uploader")
    d. Click "Create"

  ─── 4B. Enable the YouTube Data API v3 ───

    a. Go to https://console.cloud.google.com/apis/library/youtube.googleapis.com
       (make sure your new project is selected at the top)
    b. Click "Enable"

  ─── 4C. Create OAuth 2.0 Credentials ───

    a. Go to https://console.cloud.google.com/apis/credentials
    b. Click "Create Credentials" → "OAuth client ID"
     c. If it asks you to configure consent screen:
        - Choose "External" → "Create"
        - App name: "YT Shorts Upload" (or anything)
        - User support email: your email
        - Developer contact: your email
        - Click "Save and Continue" through all screens
        - No need to add scopes
        - On the "Test users" screen, click "Add Users"
        - Enter your own Google email address (the same one
          you use for YouTube) — this is CRITICAL. Without
          this, you'll get a 403 error when signing in.
        - Click "Save and Continue" → "Back to Dashboard"
     d. Back on the credentials page:
       - Application type: "Desktop app"
       - Name: "yt-shorts-script"
       - Click "Create"
    e. A popup shows your Client ID and Client Secret.
       Copy both and save them temporarily.

  ─── 4D. Run auth_setup.py on your computer ───

    This is the ONLY step that uses your local computer.
    It installs ~5 MB of Python packages and opens a browser
    for you to sign in to Google.

    a. Open a terminal/command prompt on your computer
    b. Navigate to the project folder:
         cd path/to/yt-shorts-folder
    c. Install the auth helper (one-time, ~5 MB):
         pip install google-auth-oauthlib google-auth
    d. Run the script:
         python auth_setup.py
    e. When prompted, paste your Client ID and press Enter
    f. Paste your Client Secret and press Enter
    g. A browser window will open asking you to sign in
       to Google. Sign in with your YouTube channel's Google
       account.
    h. Click "Continue" (it will say "Google hasn't verified
       this app" — this is normal for self-use, click continue)
    i. Click "Allow" to grant YouTube upload permissions
    j. Close the browser tab when it says "Authentication
       successful! You may close this tab."

    k. Go back to the terminal. You will see:
         =======================================================
         SUCCESS! Add these 3 values to GitHub Secrets from terminal:
         =======================================================
         YT_REFRESH_TOKEN: 1//0g...
         YT_CLIENT_ID: 123456789-abc.apps.googleusercontent.com
         YT_CLIENT_SECRET: G0C...
         =======================================================

    l. Keep this terminal open — you need these 3 values
       for the next step.

  ─── 4E. Add secrets to GitHub ───

    a. Go to your GitHub repository page
    b. Click "Settings" tab → "Secrets and variables" → "Actions"
    c. Click "New repository secret"

    Add these 3 secrets (one at a time):

    Secret 1:
      Name:    YT_REFRESH_TOKEN
      Secret:  Paste the long token from step 4D (starts with 1//0g...)

    Secret 2:
      Name:    YT_CLIENT_ID
      Secret:  Paste your Client ID

    Secret 3:
      Name:    YT_CLIENT_SECRET
      Secret:  Paste your Client Secret

    Secret 4:
      Name:    PEXELS_API_KEY
      Secret:  Paste your Pexels API key from Step 3E

    Secret 5 (optional — skip if you don't want background music):
      Name:    PIXABAY_API_KEY
      Secret:  Paste your Pixabay API key from Step 3F

  That's it! You never need to do this again. These tokens
  work forever until you manually revoke them.

  ─── Important notes about OAuth tokens ───

    • ✓ Refresh tokens last forever (years) — no expiry
    • ✓ No browser/cookies needed in GitHub Actions
    • ✓ Uses official YouTube Data API — never breaks
    • ✓ Only 2 API calls per video — well under free quota
    • ✓ No Playwright — faster runs, less memory
    • If you ever need to revoke access, go to:
      https://myaccount.google.com/permissions

    a. Go to your repository's main page
    b. Look for ".github" folder → click it → "workflows" folder
    c. Do you see "schedule.yml" there?
         YES → Go ahead
         NO  → Follow the fix below

  ─── If the workflow file is missing ───

    ✅ CORRECT: Stay on MAIN page → "Add file" → "Create new file"

 ____ Note: DON'T forget to set "Read and write permissions" in GitHub
      Settings → Actions → General → Workflow permissions →
       "Read and write permissions". Without this, the git push
       step fails and scripts.json never gets marked as used.

  ─── When you see the workflow ───

    a. Go to your repo → "Actions" tab
    b. Click "YouTube Shorts Automation" in the left sidebar
    c. Click "Run workflow" dropdown → green "Run workflow" button
    d. Wait and watch the live logs

  ─── What happens during the workflow ───

    1. GitHub sets up a Ubuntu Linux computer in the cloud
    2. Installs ffmpeg, ImageMagick, and fonts
    3. Installs Python and required libraries
    4. Reads the next unused script from scripts/scripts.json
    5. Uses edge-tts to create voiceover
    6. Downloads video clips + images from Pexels
    7. Downloads background music from Pixabay (if key set)
    8. Stitches everything together with MoviePy (Ken Burns zoom,
       animated captions, background music)
    9. Gets a fresh access token from your refresh token
   10. Uploads directly to YouTube via the official API
   11. Marks the script as "used" and pushes back to GitHub
   12. Cleans up all temporary files

    • "Uploaded! Video ID: yyyyyyy" and "https://youtu.be/yyyyyyyy"
    • Go to your YouTube channel to see the uploaded Short

  ─── If it fails ───

    • Read the error in the logs
    • See "TROUBLESHOOTING" section below. (OR ask any an AI about the error, provided with this GUIDE )

--> Note: sometime GitHub Actions cron schedules do not run exactly on the minute
          due to overload. Sometime it delays upto 2 hours. To bypass the heavy 
          server traffic on GitHub, never schedule your cron on a flat hour (like :00).
          Shift your minutes to a random, odd number (like :17, :34, or :41). Fewer
          people use these numbers, so your workflow will clear the queue much faster.

  COMMON MISTAKES TO AVOID (What NOT to do)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ── Workflow / GitHub Mistakes ──

  1. DON'T create the workflow file at the wrong path
     → Type .github/workflows/schedule.yml in ONE filename box.
       Do NOT click into folders first. Stay on the MAIN repo page.

  2. DON'T name the folder "github" instead of ".github"
     → The dot at the start is required. GitHub Actions ignores
       the folder without it.

  3. DON'T forget to set "Read and write permissions" in GitHub
     → Settings → Actions → General → Workflow permissions →
       "Read and write permissions". Without this, the git push
       step fails and scripts.json never gets marked as used.

  4. DON'T upload files inside a subfolder
     → When dragging your project to GitHub via "Upload files",
       drag the CONTENTS of the folder (main.py, src/, etc.),
       NOT the folder itself. Files must be at the root level.


  ── OAuth / YouTube Mistakes ──

  5. DON'T skip adding yourself as a test user in OAuth consent
     → After creating the consent screen, go to "Test users"
       and add your own email. Otherwise you get Error 403.

  6. DON'T share your refresh token, client ID, or client secret
     → Keep them in GitHub Secrets. Never paste them in code.

  7. DON'T use the AIza... or AQ... API key format
     → Those are for Google AI Studio (Gemini API), not for
       YouTube. This system doesn't need any Gemini API key.

  ── scripts.json Mistakes ──

  8. DON'T put scripts as a single object {} instead of array []
      → MUST be a JSON array: [ {...}, {...}, ... ]

  9. DON'T change the JSON field names in scripts.json
      → Must match exactly: title, description, tags,
        voiceover_script, visual_keywords, used.

  10. DON'T forget to REMOVE "]" and ADD "," when merging batches
      → Follow the merge instructions below.
    

    Step 1: Open your existing scripts/scripts.json from GitHub
            The end of the file looks like:
              },
              {
                ...last script...
              }
            ]

    Step 2: REMOVE the closing bracket "]" at the very end.
            Now the file ends with the last script's } and nothing after.

    Step 3: ADD a comma "," after the last script's closing }
            It should now look like this at the end:
              }
            ]
              ↓ Remove ]
              ↓ Add ,
            },
            

    Step 4: Open the NEW batch from the AI. It starts with:
            [
              {
                ...first new script...
              }

    Step 5: REMOVE the opening bracket "[" from the new batch.
            Now it starts directly with {
              {
                ...first new script...
              }

    Step 6: Paste the new batch (without the [) directly AFTER the
            comma you added in Step 3. The result:

            },
            {
              ...first new script of batch 2...
            },
            {
              ...second new script...
            }
            ]

    Step 7: Add the closing bracket "]" at the very end.

    Step 8: Save and upload to GitHub.

    ─── Example of the merge ───

    Existing file (end part):
      {
        "id": 10,
        "title": "Last script of batch 1",
        ...
        "used": true
      }
    ]

    Step 1: Remove ]
    Step 2: Add comma after }
      {
        "id": 10,
        ...
        "used": true
      },
    Step 3: Take new batch, remove its opening [
    Step 4: Paste after the comma:
      {
        "id": 10,
        ...
        "used": true
      },
      {
        "id": 11,
        "title": "First script of batch 2",
        ...
        "used": false
      },
      ...
      {
        "id": 20,
        ...
        "used": false
      }
    ]
    Step 5: Add ] at the end

    Done. The file now has 20 scripts (10 used + 10 unused)

  11. DON'T leave placeholder content in scripts.json
      → Delete the example entry and paste your real AI output

  ── Video / Pexels Mistakes ──


  12. DON'T change the video resolution or codec settings
      → 1080×1920 at 30fps with h.264 is what YouTube Shorts
        expects. Changing these causes upload failures.

  ── General Mistakes ──

  13. DON'T run this locally on Windows — it will probably fail
      → Designed for GitHub's Ubuntu runners. ImageMagick paths
        and ffmpeg are different on Windows. Use WSL if needed.

  14. DON'T modify the workflow YAML file without a proper editor
      → YAML is sensitive to spaces vs tabs. Use VS Code,
        Notepad++, etc. Not regular Notepad.

  15. DON'T worry about the "Node.js 20 is deprecated" warning
      → It's just a warning, not an error. Completely harmless.


  16. DON'T run out of GitHub Actions free minutes
      → Free accounts get 2000-3000 min/month. Each run is
        ~ less the 10 min. At 2/day you use ~600 min/month. Fine.
        But don't manually trigger too many extra runs.

Auto_You supports premium human-like voice narration using Google
Gemini 3.1 Flash TTS Preview.

If you don't do this step, the system will automatically fall back
to Microsoft's free edge-tts narrator voice (en-US-AndrewNeural).

  ─── 1. Get a Gemini API Key ───
  a. Go to Google AI Studio: https://aistudio.google.com/
  b. Sign in with any Google account.
  c. Click the "Get API Key" button.
  d. Click "Create API Key" and copy it.

  ─── 2. Add to GitHub Secrets ───
  a. Go to your repository on GitHub.
  b. Go to Settings → Secrets and variables → Actions.
  c. Click "New repository secret".
  d. Name: GEMINI_API_KEY
  e. Value: Paste your Gemini API key.
  f. Click "Add secret".

  ─── 3. Configure Voices and Models (Optional) ───
  By default, the system uses the "Aoede" voice and the
  "gemini-3.1-flash-tts-preview" model.
  If you want to use a different voice (like "Leda", "Fenrir",
  "Kore", "Aoede", "Puck", or "Charon"), you can add a Repository
  Variable in GitHub:
  a. Go to Settings → Secrets and variables → Actions.
  b. Click the "Variables" tab next to Secrets.
  c. Click "New repository variable".
  d. Name: TTS_VOICE
  e. Value: Leda (or another supported voice)
  f. Save it. You can do the same for TTS_MODEL if needed.

  ─── 4. Advanced Script Customization (Optional) ───
  In your scripts.json file, you can optionally add "audio_profile"
  and "directors_note" fields to control the tone, accent, pacing,
  or emotional style of the voice. Example:
  
  {
    "title": "Quantum Levitation",
    "voiceover_script": "Imagine a world where objects float... [sfx: chime]",
    "audio_profile": "A deep, resonant narrator of mysteries.",
    "directors_note": "Style: Vocal Smile. Pace: The Drift. Accent: American.",
    "visual_keywords": ["quantum", "magnet"],
    "used": false
  }

  The system will parse and strip any [sfx: query] tags in your script
  text, automatically search Openverse CC0 for matching sound effects,
  and mix them at the exact timestamp!
