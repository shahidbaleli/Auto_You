Thanks for looking at Auto_You — contributions are welcome.

A few simple guidelines to make collaboration smooth:

1. Fork the repository and open a pull request from your fork.
2. Use clear, small commits. Prefer a single focused change per PR.
3. Write a short description of what your change does and why.
4. Follow the existing code style (clear variable names, simple functions).
5. If your change adds or updates behavior, include a short example or test.
6. Label your PR with relevant keywords (bug, feature, docs).

If you're unsure where to start, open an issue describing the problem or idea and we'll discuss it.
